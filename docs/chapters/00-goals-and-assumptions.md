# 00. 이 가이드의 목표와 전제

## 학습 목표

이 책의 목표는 Go를 "문법 암기"가 아니라 "실무 서비스 구현" 관점으로 익히는 것입니다.

학습은 다음 순서로 진행합니다.

1. 문법과 타입 시스템을 이해한다.
2. 표준 라이브러리로 서버를 만들 수 있다.
3. 동시성 도구(goroutine/channel/context)를 안전하게 쓴다.
4. DB/캐시/메시지를 포함한 백엔드 서비스를 구현한다.
5. 테스트, 성능 점검, 배포/운영까지 연결한다.

최종적으로 다음을 스스로 할 수 있어야 합니다.

1. Java 코드베이스를 기준으로 Go 서비스 구조를 설계한다.
2. REST API를 작성하고 데이터 계층과 연동한다.
3. 장애 대응을 고려해 타임아웃/취소/리트라이를 적용한다.
4. 측정 가능한 방식으로 성능을 개선한다.
5. 운영 가능한 빌드/배포 파이프라인을 준비한다.

## 이 책의 전제

이 책은 아래 전제를 두고 작성합니다.

1. 독자는 Java 기본 문법, HTTP, SQL 기초를 알고 있다.
2. 프레임워크 의존보다 표준 라이브러리 이해를 우선한다.
3. 예제는 "작동"뿐 아니라 "운영 가능성"을 기준으로 설명한다.
4. 모든 코드는 Go Modules 기준으로 관리한다.

다루지 않는 범위도 명확히 합니다.

1. GUI/데스크톱 앱 개발
2. Go 언어 명세 전체의 완전한 해설
3. 특정 클라우드 벤더 전용 기능 심화

## 권장 개발 환경

아래 환경을 기준으로 진행합니다.

1. Go: 최신 안정(stable) 버전 사용
2. 모듈: `go mod` 기반 의존성 관리
3. IDE: VS Code(Go 확장) 또는 GoLand
4. 포맷터: `gofmt`, `goimports`
5. 린터: `golangci-lint`
6. 테스트: `go test` (unit/integration/benchmark)

버전 정책은 다음처럼 가져갑니다.

1. 책의 예제는 최소 지원 버전(Minimum Supported Go Version)을 명시한다.
2. CI에서는 최소 버전과 최신 안정 버전을 함께 검증한다.
3. 로컬 개발 환경은 최신 안정 버전을 권장한다.

로컬에서 먼저 확인할 항목:

```bash
go version
go env GOPATH GOMOD GOCACHE
```

## 예제 프로젝트 소개

책 전체에서 하나의 예제 서비스를 점진적으로 완성합니다.

- 프로젝트 이름(가칭): `go-commerce-api`
- 목표: 주문/결제 흐름을 단순화한 REST API
- 구성: HTTP API + DB + 캐시 + 메시지 이벤트 + 백그라운드 워커

예제에서 다룰 핵심 기능:

1. 상품/주문 조회 및 생성 API
2. 트랜잭션 기반 주문 저장
3. Redis 캐시 조회/무효화
4. 이벤트 발행(예: `order.created`)
5. 컨슈머 워커의 재시도/실패 처리

권장 기술 스택:

1. HTTP: `net/http`
2. DB: PostgreSQL (`database/sql`)
3. Cache: Redis
4. Message: NATS 또는 Kafka 중 하나 선택
5. 관측성(Observability): 구조적 로그 + 메트릭 + 트레이스 기본

초기 디렉터리 예시는 아래와 같습니다.

```text
go-commerce-api/
├─ cmd/
│  └─ api/
├─ internal/
│  ├─ domain/
│  ├─ service/
│  ├─ repository/
│  ├─ transport/http/
│  └─ worker/
├─ pkg/
├─ migrations/
├─ deployments/
└─ test/
```

## 학습 방법

각 장은 같은 루프로 학습합니다.

1. 개념: Java와의 차이를 먼저 확인
2. 코드: 작은 단위 예제 실행
3. 확장: 예제 프로젝트에 기능 반영
4. 검증: 테스트/벤치마크/프로파일링

진행 중 막힐 때는 다음을 우선 점검하세요.

1. 컴파일 에러 메시지를 끝까지 읽었는가
2. `go test ./...`가 현재 상태에서 통과하는가
3. 타임아웃/컨텍스트 전파가 빠진 지점은 없는가
4. 에러를 감추지 않고 호출자까지 전달하는가

## 요약

1. 이 책은 문법 학습을 넘어 서비스 개발/운영까지 연결하는 것을 목표로 한다.
2. 표준 라이브러리 중심의 실무적 접근을 전제로 한다.
3. 예제 프로젝트(`go-commerce-api`)를 통해 HTTP, DB, 캐시, 메시징을 단계적으로 완성한다.
4. 각 장은 개념 -> 코드 -> 확장 -> 검증 루프로 학습한다.
